# 04 · 能力路线图

> 标注每个能力撑哪条故事：**[GRAPH]** 多跳图谱（→ Agentic-RL 可验证奖励）· **[RAG]** 设定/梗检索 · **[REC]** 推荐 · **[PERSONAL]** 个人/社交黏性。
> 难度：易 / 中 / 难。

## Tier 1 —— v0.1 首刀（脊/面/算最小闭环都点亮）

| 能力 | 说明 | 数据 | 故事 | 难度 |
|---|---|---|---|---|
| **声优 filmography + 同台网络** | "这个声优配过的所有角色/番"，按年份/角色重要度过滤；"X 和 Y 同台过的番" | Bangumi 图谱 | [GRAPH] 可验证 | 易–中 |
| **系列时间线梳理** ⭐ | 从关联边+air date 生成系列的发行/时间线顺序（**客观可校验**） | 图谱+air date | **[GRAPH] 旗舰可验证任务** | 中–难 |
| **口味画像 / 二次元人格** | 类型/制作/年代/声优分布，"你是声优厨/作画党/剧情党"，可分享 | 用户收藏+图谱 | [REC]+[PERSONAL] | 易–中 |

> 选这三个的理由：一个纯图谱多跳（声优网络）、一个旗舰可验证（系列时间线，撑 RL 故事）、一个推荐+黏性（口味画像，撑新技能与传播）。
> 注：**"防剧透最佳补番顺序"不放 v0**——它含剧透判断+主观性，做不成干净真值，不该当可验证 benchmark 任务，降级为 Tier-2 产品功能（仍要做）。

## RL 重点任务（撑 Agentic-RL 可验证奖励）

优先做有客观真值、能 exact-match/precision-recall 打分的图谱遍历：
**系列时间线（发行顺序）、声优同台/Bacon 数路径、跨媒体 IP 链接、声优 filmography 完整性**。
RAG 类（考据/梗）承载"wow"，个人/社交类（画像/affinity/Wrapped）承载留存与传播。

## 设计原则：校准化拒答（一等公民）

agent 必须**知道自己的数据边界**，超出时**诚实说"查不到/不在我的数据范围"，而不是编**。
- 仍属结构性边界、不强行作答的：**资讯/最新公告**（Bangumi 非资讯源、有编辑延迟）、**BD 销量/周边价格**、**同人二创**。
- "在哪看"不靠猜，只给**外链**（见 [02 §3.1](02-data-sources.md)）。
- **校准化拒答纳入评测**（见 [05](05-evaluation.md)）：该拒则拒 = 正确；这与 verifier/可验证理念一脉相承，是抑制幻觉的关键。

## 版本与能力的映射

- **v0.x**：Tier 1 全部 + 至少 2 个 Tier 2（建议"梗/考据 QA"打通 RAG 链路、"季番分诊"打通推荐链路）。
- **v1.x**：补齐 Tier 2 + Tier 3 选做。
- **v2.x**：把"补番顺序/声优路径"做成可验证 benchmark → Agentic-RL；把推荐偏好做成 DPO/GRPO。
