# 00 · 项目愿景与定位

## 一句话

基于 Bangumi API 与萌娘百科/中文维基，构建一个能理解用户偏好、在二次元知识图谱上做多跳检索与可解释推荐、规划追番路径、管理收藏进度，并通过可验证评测持续迭代的 **ACGN 垂直 Agent**。

## 为什么做这个（设计动机）

1. **兴趣驱动可持续**：作者是二次元爱好者，愿意长期迭代——这是个人项目能做深的前提。
2. **一个项目覆盖四类能力**（见下"四层结构"），同时服务"Agent 后训练算法 + Agent 工程 + 推荐系统"三条求职线。
3. **数据源天然形成"结构化 vs 非结构化"的张力**：Bangumi 给可验证的图谱事实，萌娘给设定/梗的长文本。两者交叉校验、抑制幻觉，正是有分量的核心难题。

## 四层结构

> 设计哲学贯穿全项目：**先最小、先手搓，只有当真实需求逼近时才引入框架/基建**（DIY→LangGraph，pgvector→Qdrant，进程内→异步 worker）。每一次"升级"都记录其触发的真实需求，本身就是工程叙事。

### 脊（矛 · 算法护城河）
Bangumi 是一张真正的四实体知识图谱（番/角色/人物/章节），边带上下文、可程序化校验。
→ 在其上构建**多跳可验证知识 Agent**：问题拆解 → 图谱多跳遍历 → 萌娘 RAG 补设定 → 综合作答，事实部分对真值边自动判分。
→ 这套"可验证奖励"是 Agentic-RL 最难的部分，而本域**免费提供真值**。后期复用作者已有的 harness/verifier/轨迹回收，做 SFT 冷启动 → GRPO/DAPO 后训练。

### 面（广度 · 覆盖 Agent 开发能力面）
不止"问答"，而是一组能力面（详见 [04-capabilities](04-capabilities.md)）：
声优 filmography 与同台网络、staff/制作公司谱系发现、补番顺序/系列时间线、口味画像、季番速览与追番分诊、跨媒体 IP 追踪、考据/梗问答、防剧透问答、好友口味对比……
→ 天然覆盖 RAG、Memory、Tool-Use、MCP/Skills、Agent 编排、评测、可观测、流式、安全、API 服务化。

### 工（盾 · 工程纵深）
手搓 ReAct/Plan-Execute 核心 → 再用 LangGraph 重写同一 agent → 多维对比；
Next.js 全栈（chat + 推荐卡片 + trace 面板）；
Postgres+pgvector → Qdrant；Redis（缓存/记忆/限流/队列/排行榜）；异步 worker + pub/sub SSE。

### 算（新技能 · 推荐系统 + RL）
作者强于 LLM/RL、弱于推荐——本项目补这块：
评测脚手架+流行度基线 → 内容/itemCF → ALS/BPR 矩阵分解 → **MF 召回 + LambdaMART 重排** → LLM 特征解决冷启动 → **DPO/GRPO 用用户自己的 accept/reject 偏好后训练**。
最后一跳又回到作者的 DPO/GRPO 主场——recsys 与 LLM 后训练在此收束。

## 目标用户

- **核心**：硬核二次元/动漫爱好者（追声优、考据、补番、刷分党）。
- **次要**：想要"按我口味推荐+解释为什么"的普通追番用户。

## 覆盖范围：ACGN 全类型（一等原则）

"ACGN = Anime/Comic/Game/Novel"，只覆盖动画名不副实。**所有模块（图谱工具、RAG、口味画像、评测、推荐）最终都要覆盖全类型**；当前 anime 优先、**默认 anime 但参数可配**，禁止再新增 anime-only 硬编码。类型映射与子类细分见 [02 §1.7](02-data-sources.md)（comic/novel 同属 Bangumi book、galgame 属 game，需 tag/平台子分）。

## 非目标（明确不做）

- 不做通用聊天/陪聊角色扮演（持久人设评测主观，偏离算法主线）。
- 不与 CV 博士在纯视觉/多模态研究上正面竞争；多模态（截图识番）仅作可选彩头，且优先包装现成服务（trace.moe）。
- 不碰 Kafka/K8s/Milvus 级"假复杂度"（对本体量不合理）。

## 简历叙事（做完后能写出的话）

- "手搓 ReAct/Plan-and-Execute Agent Runtime（工具注册、状态、记忆、两级 Verifier、SSE 流式、轨迹回放），后以 LangGraph 重写并在 8 个维度做工程对比。"
- "构建可验证多跳评测集（基于 Bangumi 图谱真值边自动生成），评估 tool-call 准确率、Recall@K、引用正确率、P95 延迟。"
- "实现推荐 recall→rank 漏斗（MF 召回 + LambdaMART 重排），温度分割与流行度去偏评测，NDCG@10 较基线提升 X%。"
- "将 DPO/GRPO 用于推荐重排与多跳 Agent 后训练，以用户自身偏好/图谱可验证奖励提升个性化与任务成功率。"
